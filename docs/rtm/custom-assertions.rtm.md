## Traceability summary

- ACs covered: 5 / 5 (plus the always-on AXE requirement)
- Test cases: 24 unique cases × 4 browsers = 96 executions (P:4 N:9 B:3 D:8 A11y:3 NF:1 — several
  cases are jointly negative + data-driven or boundary + data-driven, tagged accordingly)
- Notable finding (hydration race): mutating `[data-testid="order-price"]` / `order-status` text
  via `page.evaluate()` immediately after `page.goto()` resolves races the app's React hydration —
  the mutation gets silently reverted once hydration reconciles. Fixed by waiting for the order
  heading plus a 300ms settle in `beforeEach` before any DOM mutation runs (see
  `tests/custom-assertions/custom-assertions.spec.ts` beforeEach comment).
- Notable finding (AC-3): the order-status badge's visible text is CSS-`capitalize`d but its
  underlying DOM text node already matches `data-status` exactly in the live app — the spec proves
  the matcher is attribute-driven (not text-driven) by mutating only the visible text node and
  confirming the matcher still passes against the untouched attribute.
- Every POM interactive element exercised by ≥1 case: yes (orderPrice, orderStatus,
  advanceStatusButton, star1–star5 via star4, ratingValue all asserted)
- Open defects: 0
