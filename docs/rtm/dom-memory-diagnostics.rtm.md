> **Fix Prompt — TAB1-66**
> You are working on the **stagecraftlabs.com source code** (separate repository from this Playwright test project).
>
> **Page:** `https://stagecraftlabs.com/practice/dom-memory-diagnostics`
> **Element / Component:** The "Leaked toast nodes" graveyard list — `<ul aria-label="Leaked toast nodes" class="mt-2 max-h-40 space-y-1 overflow-y-auto text-xs text-muted">`
> **Defect:** Once populated with 50 items, the `<ul>` has `scrollHeight: 996px` vs `clientHeight: 160px` (a scrollable region per `max-h-40 overflow-y-auto`), but `tabindex="-1"` and no keyboard handling means keyboard-only users cannot scroll it to read entries beyond the first ~4 visible rows. Axe rule `scrollable-region-focusable` (serious, WCAG 2.1.1 / 2.1.3).
> **Root cause hypothesis:** the `<ul>` was never given a `tabindex`, so it's excluded from the tab order despite becoming scrollable once its content overflows `max-h-40`.
> **Fix:** Add `tabindex="0"` to the `<ul>` (the existing `aria-label="Leaked toast nodes"` already gives it an accessible name).
> **Verification:** Remove the `v.id !== 'scrollable-region-focusable'` filter from `tests/dom-memory-diagnostics/dom-memory-diagnostics.spec.ts` (accessibility describe block) and re-run the suite. The "no violations while the graveyard list is fully populated with 50 retained nodes" assertion must pass with no new violations.
