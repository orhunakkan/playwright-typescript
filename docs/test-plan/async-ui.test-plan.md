## 1. Scope

**In scope**

- Loading skeleton: visible immediately on load-trigger, disappears when real content replaces it
- Delayed news articles: asserted with `waitFor` / `toBeVisible` â€” zero `page.waitForTimeout()` calls
- Stock price ticker: auto-updates every 2 s; asserted with `expect.poll()` until target value reached
- Error state: triggered by "Load with error" button; error message asserted visible
- Transient toast: appears for 800 ms; text asserted before auto-dismissal
- Async-settling guarantee: no premature assertions that could cause flakiness
- Accessibility (axe) on all UI states: initial, skeleton, loaded content, error, toast-visible
- Performance budget on initial page load

**Out of scope**

- Backend API implementation or network stubbing (lab is client-side rendered)
- Persistent data storage or user sessions
- Load / stress testing
- Internationalization / locale variants
