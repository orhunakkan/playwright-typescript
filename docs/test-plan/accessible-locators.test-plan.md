## 1. Scope

**In scope**

- Semantic locator coverage: all element queries use `getByRole`, `getByLabel`, `getByAltText`, or `getByText` — zero CSS selectors, test IDs, or XPath
- "Available" book filtering: assertions scoped with `locator.filter` to the "Available" subset; no hardcoded row indices
- Genre filter button: click a genre, assert all visible book cards belong to that genre and non-matching genres are absent
- Book cover image: located via `getByAltText`; confirmed visible with `toBeVisible`
- Chained card-scoped locators: button clicks targeted to a specific book card via chained locators, not page-wide selectors
- Search input: filtering the book list by query; result count updates correctly; empty / no-match edge cases
- Accessibility (axe) on all UI states: initial load, genre-filtered view, and search-results view
- Performance budget on initial page load

**Out of scope**

- Backend implementation or API mocking (read-only catalogue page)
- Persistent user sessions or authentication
- Load / stress testing
- Internationalization / locale variants
