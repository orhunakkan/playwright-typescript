## 1. Scope

**In scope**

- Subscribe button enable/disable gating on the required fields (name, email, topic, frequency, terms)
- Inline field validation on blur (name length, email format, category required)
- `aria-invalid` / `aria-describedby` accessibility contract on invalid fields
- Successful submission confirmation + form reset
- Optional profile-picture file upload
- Accessibility (axe) on load, error, and success states
- Performance budget on initial load

**Out of scope**

- Backend persistence of the subscription (no API/DB assertions — the lab is client-side)
- Email deliverability / real mail sending
- Internationalization / localization of validation messages
- Load and stress testing
