## Overview

Capture full-page screenshots of every page in the application to serve as visual baselines. These baselines are used in future runs to detect unintended visual changes. All pages are covered in a single task since the approach is identical for each.

## Task

Navigate to each of the following pages and capture a full-page screenshot:

- Web form
- Index (home page)
- Navigation — first variant
- Navigation — third variant
- Dropdown menu
- Mouse over
- Drag and drop
- Draw in canvas
- Loading images
- Slow calculator
- Long page
- Infinite scroll
- Shadow DOM
- Cookies
- Frames (frameset)
- IFrames
- Dialog boxes
- Web storage
- Geolocation
- Notifications
- Get user media
- Multilanguage
- Console logs
- Random calculator
- Download
- Data types
- A/B testing — Variation A (inject variation A content before capturing)
- A/B testing — Variation B (inject variation B content before capturing)

For the A/B testing screenshots, the variation content must be injected into the page before capturing so each screenshot represents a deterministic state, not a random one.
